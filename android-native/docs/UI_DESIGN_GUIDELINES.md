# 原生 Android UI 设计规范（v1）

## 1. 设计目标

- 强化移动端“快速上传与快速检索”效率。
- 保持高信息密度同时确保单手操作舒适。
- 建立可扩展 Design Token 与组件层级。

## 2. 视觉语言

- 主色：`#3B82F6`
- 辅助色：`#6366F1`
- 浅色背景：`#F8FAFC`
- 深色背景：`#020617`
- 文本层级：标题 > 正文 > 辅助信息，最小触控热区 44dp。

## 3. 间距与圆角

- 页面外边距：20dp
- 区块间距：16dp
- 列表项间距：8dp
- 卡片圆角：14dp
- 按钮圆角：12dp

## 4. 关键页面交互

- 登录页：单主动作按钮，错误反馈即时可见。
- 文件页：顶部筛选 + 列表信息密度优先 + 空态引导上传。
- 上传页：任务中心化展示，支持暂停与失败重试。
- 预览页：沉浸内容优先，工具栏提供下载/分享/删除快捷操作。
- 设置页：账号、安全、主题、存储按分组卡片展示。

## 5. 状态规范

- 每个页面必须覆盖 6 态：初始、加载、成功、空态、错误、离线。
- 异步操作必须有可见反馈，禁止无状态等待。
- 错误文案包含原因与下一步操作建议。

## 6. 可访问性与体验

- 支持动态字体缩放。
- 支持深色模式且对比度合规。
- 关键操作提供震动或视觉确认反馈。
- 动效时长控制在 160ms~260ms 区间。

# C-005: CI 覆盖率门槛

## 约束内容
CI 覆盖率门槛使用真实全局基线守门：`cargo-llvm-cov --fail-under-lines 23`。
当前本地实测全局行覆盖率为 `2542 / 11012 = 23.08%`，因此 23% 是
现阶段必须保持的最低阻塞线；90% 是持续补测后的目标线，不能在未达成
前写成当前硬门槛。

## 触发原因
确保代码质量，强制测试覆盖不倒退，同时避免把未来目标写成永远失败的
当前 CI 红线。

## 生效日期
2026-05-02

## 关联文件
- `.github/workflows/ci.yml`
- `docs/constraints/ci-pipeline.md`

## 例外条件
任何低于 23% 的下调都必须先获得人类明确批准。后续每次补覆盖达到新的
稳定基线后，只允许上调该门槛。

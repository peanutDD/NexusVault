# C-051 Auto Review Comment Checklist E2E

## Constraint

The fully automated review loop must have end-to-end coverage proving that `pr-auto-fix` posts a PR comment with the `Medium/Medium+/High/Critical 对应状态` checklist.

The captured comment must include:

- the one-to-one issue status table
- at least one `已解决` marker when a review issue was fixed
- at least one `未解决` marker when a review issue remains pending
- the resolved and pending issue descriptions

## Rationale

JSON output and `docs/auto-review-ledger.md` are not enough to prove the reviewer-facing PR comment path works. The user-facing promise is that every automated PR review run leaves a visible issue checklist in the conversation, so tests must exercise `pr-auto-fix` through its `gh pr comment` boundary.

## Enforcement

- E2E coverage: `pr_auto_fix_posts_issue_status_checklist_comment_in_dry_run`.
- The test must use a fake `gh` executable to capture the exact comment body generated by the automated flow.

#### [BE-002] 登录错误次数限制 / 验证码机制
**优先级：** 高（安全加固）  
**描述：**  
实现登录保护机制，防止暴力破解：
- 记录每个 IP / 邮箱的登录失败次数
- 达到阈值（如 5 次）后要求验证码或临时锁定
- 使用内存缓存（如 moka）记录失败次数，设置过期时间
- 提供验证码生成接口（可选：图形验证码或邮件验证码）

**验收标准：**
- [ ] 登录失败次数限制生效
- [ ] 验证码机制正常工作
- [ ] 锁定时间可配置
- [ ] 不影响正常用户登录体验

#### [BE-003] 完整的邮箱验证 / 找回密码流程
**优先级：** 中  
**描述：**  
实现完整的邮箱验证和密码找回功能：
- **邮箱验证**：
  - 注册时发送验证邮件
  - 验证链接包含一次性 token 和过期时间（如 24 小时）
  - 验证后标记邮箱为已验证
- **找回密码**：
  - 提供"忘记密码"接口，发送重置链接
  - 重置链接包含一次性 token 和过期时间
  - 重置密码后使所有现有 token 失效

**验收标准：**
- [ ] 注册邮箱验证流程完整
- [ ] 找回密码流程完整
- [ ] Token 过期机制正常
- [ ] 邮件发送成功（测试环境可用日志模拟）
